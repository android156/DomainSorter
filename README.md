# DomainSorterBot

Telegram-бот для накопления, дедупликации и сортировки списков доменов и IP-маршрутов.
Бот работает в Telegram под именем **@DomainSorterBot**.

---

## Для чего нужен бот

Когда у вас накапливается много `.txt`-файлов со списками доменов или `.bat`-файлов с IP-маршрутами из разных источников, бот решает три задачи:

1. **Собирает** все данные из произвольного числа файлов в единый список.
2. **Дедуплицирует** — одинаковые домены или маршруты добавляются только один раз, даже если они встречаются в нескольких файлах.
3. **Сортирует и разбивает** результат на удобные части (чанки) нужного размера и отдаёт вам готовые файлы.

Каждый пользователь работает со своим изолированным списком — данные других пользователей недоступны.

---

## Быстрый старт

1. Найдите бота в Telegram: **@DomainSorterBot**
2. Нажмите **Start** или отправьте `/start`
3. Внизу экрана появится постоянная клавиатура с кнопками
4. Отправляйте боту файлы — он обработает их автоматически
5. Когда все файлы загружены, нажмите **📤 Выгрузить** — бот пришлёт готовые файлы и очистит список

## Форматы входных файлов

### Файлы доменов (`.txt`)

Обычный текстовый файл, по одному домену на строку:

```
google.com
mail.google.com
example.org
# эта строка — комментарий, будет проигнорирована
yandex.ru
```

- Строки, начинающиеся с `#`, игнорируются
- Пустые строки игнорируются
- Регистр не важен: все домены приводятся к нижнему регистру
- Дубликаты (в пределах одного пользователя) не добавляются

### Файлы IP-маршрутов (`.bat`)

Windows-совместимый bat-файл с командами маршрутизации:

```bat
route ADD 1.2.3.0 MASK 255.255.255.0 0.0.0.0
route ADD 10.0.0.0 MASK 255.0.0.0 0.0.0.0
route ADD 2001:db8:: MASK ffff:: 0.0.0.0
```

- Поддерживаются IPv4 и IPv6
- Формат строки: `route ADD <ip> MASK <маска> 0.0.0.0` (регистр не важен)
- Строки другого формата молча пропускаются
- Дубликаты по паре `(ip, маска)` не добавляются
- Оригинальный формат строки сохраняется при экспорте

## Кнопки и команды

### Постоянная клавиатура (всегда видна внизу)

| Кнопка | Действие |
|---|---|
| 📊 Статус | Показывает количество доменов и маршрутов, текущие настройки |
| 📤 Выгрузить | Экспортирует все списки файлами; после успешной отправки данные очищаются |
| 🔤 Сортировка: abc | Переключает сортировку доменов на алфавитную |
| 🌐 Сортировка: domain | Переключает сортировку доменов на «по TLD» |
| ❓ Помощь | Показывает справку по всем командам |

### Inline-кнопки под сообщением статуса

После команды `/status` под сообщением появляются кнопки:

```
[ 🔤 abc ✓ ]  [ 🌐 domain ]
[ 📤 Выгрузить список ]
```

- Активный режим сортировки помечен знаком **✓**
- Нажатие на другой режим мгновенно переключает сортировку и обновляет сообщение на месте (без нового сообщения)
- Нажатие «Выгрузить список» запускает экспорт прямо из статуса

### Inline-кнопка после загрузки файла

После успешной загрузки каждого файла появляется кнопка:

```
[ 📊 Показать статус ]
```

Позволяет быстро проверить текущий счётчик без отдельной команды.

### Команды

| Команда | Описание |
|---|---|
| `/start` | Приветствие + справка + включение клавиатуры |
| `/help` | Справка по всем командам |
| `/status` | Состояние: кол-во доменов и маршрутов, настройки |
| `/set_sort abc` | Алфавитная сортировка доменов (по умолчанию) |
| `/set_sort domain` | Сортировка доменов по TLD |
| `/set_list_len N` | Макс. доменов в одном файле (по умолчанию 300) |
| `/set_ip_list_len N` | Макс. IP-маршрутов в одном файле (по умолчанию 1000) |
| `/get_list` | Экспорт списков (имя из первого загруженного файла) |
| `/get_list имя` | Экспорт с указанным именем файлов |

## Режимы сортировки доменов

### `abc` — алфавитная (по умолчанию)

Обычная сортировка по строке слева направо:

```
api.example.com
example.com
mail.google.com
www.google.com
yandex.ru
```

### `domain` — по TLD (правая часть первая)

Домены сортируются справа налево: сначала по TLD, затем по домену второго уровня, затем по поддоменам. Все домены одной зоны окажутся рядом:

```
example.com
api.example.com
mail.google.com
www.google.com
yandex.ru
```

## Структура экспортируемых файлов

При выгрузке бот отправляет файлы в следующем порядке:

1. **Полный список** (`имя_all.txt` / `имя_all.bat`) — все записи в одном файле
2. **Чанки** (`имя.txt`, `имя-2.txt`, `имя-3.txt` …) — список, разбитый на части

Имя файла берётся из имени первого загруженного файла (без расширения). Можно указать своё имя через `/get_list моёимя`.

**Пример** при 750 доменах с лимитом 300:
```
mylist_all.txt   — 750 доменов
mylist.txt       — домены 1–300
mylist-2.txt     — домены 301–600
mylist-3.txt     — домены 601–750
```

После успешной отправки все данные и настройки имён файлов автоматически очищаются.

## Технические детали

### Стек

| Компонент | Технология |
|---|---|
| Язык | Python 3.11 |
| Telegram API | aiogram 3.x (async) |
| База данных | SQLite (через aiosqlite) |
| Переменные окружения | python-dotenv |

### Структура проекта

```
telegram-bot/
├── bot.py          — точка входа; все обработчики сообщений и callback
├── keyboards.py    — построители клавиатур (reply и inline)
├── database.py     — все операции с SQLite (изолировано по user_id)
├── processor.py    — парсинг .txt и .bat файлов; чистая логика без I/O
├── utils.py        — разбивка списков на чанки; формирование имён файлов
├── requirements.txt
└── bot_data.db     — SQLite база (создаётся автоматически при первом запуске)
```

### Схема базы данных

**`settings`** — настройки пользователя:

| Поле | Тип | По умолчанию | Описание |
|---|---|---|---|
| `user_id` | INTEGER PK | — | Telegram user ID |
| `sort_mode` | TEXT | `abc` | Режим сортировки: `abc` или `domain` |
| `list_len` | INTEGER | `300` | Макс. доменов в одном файле |
| `ip_list_len` | INTEGER | `1000` | Макс. маршрутов в одном файле |
| `first_domain_filename` | TEXT | NULL | Имя первого загруженного .txt |
| `first_ip_filename` | TEXT | NULL | Имя первого загруженного .bat |

**`domain_items`** — хранилище доменов:

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Авто-инкремент |
| `user_id` | INTEGER | Владелец записи |
| `domain` | TEXT | Доменное имя (уникально по user_id) |

**`ip_items`** — хранилище IP-маршрутов:

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Авто-инкремент |
| `user_id` | INTEGER | Владелец записи |
| `ip` | TEXT | IP-адрес |
| `mask` | TEXT | Маска подсети |
| `original_line` | TEXT | Исходная строка `route ADD …` (сохраняется для вывода) |
| `sort_key` | BLOB | Байтовый ключ для сортировки (IPv4 < IPv6) |

### Сортировка IP-маршрутов

IP-адреса сортируются по числовому значению, а не лексикографически. IPv4-адреса всегда идут перед IPv6:

- IPv4: ключ `\x00` + 4 байта адреса (итого 5 байт)
- IPv6: ключ `\x01` + 16 байт адреса (итого 17 байт)

### Запуск (для разработчиков)

```bash
# 1. Установить зависимости
pip install -r telegram-bot/requirements.txt

# 2. Задать токен бота
export TELEGRAM_BOT_TOKEN=ваш_токен
# или создать файл telegram-bot/.env:
# TELEGRAM_BOT_TOKEN=ваш_токен

# 3. Запустить
cd telegram-bot
python3 bot.py
```

На Replit бот запускается автоматически через workflow «Telegram Bot».
